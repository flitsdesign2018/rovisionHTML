SVG with JavaScript Recommended

Our brand new SVG-based framework that gives you all the benefits of using SVG icons on the web without all of the hassle. We’ve kept the syntax and steps simple, and built tools for sizing, placement, and styling on top of that.

Get Started First!

If you didn't set up Font Awesome in your project yet, you'll want to do that first. Mmmkay?

How it Works

SVGs are awesome, but they can be a pain. Our new SVG with JS framework replaces your Font Awesome icon elements (e.g. <i class="fas fa-camera-retro"></i>) with SVG directly in the page. AND our SVG icons automatically inherit CSS size and color. This means you get Font Awesome-level ease of use without the usual hassle of SVG. (Also note we’re automatically making icons accessible. Make sure to read about our auto-accessibility.)

Icons will load asynchronously in the page. This has some benefits and some drawbacks. The primary benefit to loading this way is that it does not block the page from rendering while it waits on the icons to load. However, this can also be viewed as a negative because once the icons are loaded they can cause the layout to shift.

Basic Usage

You can place Font Awesome icons just about anywhere using a CSS style prefix and the icon’s name. Font Awesome is designed to be used with inline elements (we like the <i> tag for brevity, but using a <span> is more semantically correct).

<i class="fas fa-camera-retro"></i>

Font Awesome icons automatically inherit CSS size and color. This means they blend in with text inline wherever you put them.

<div style="font-size:3em; color:Tomato">
  <i class="fas fa-camera-retro"></i>
</div>

Icon animation wobble foibles

We've worked hard to keep icons perfectly centered when they are spinning or pulsing. However, we've seen issues where icons in desktop Safari still have a wobble. This appears to be a bug in Safari.

One way that you might be able to work around this is by using display: block; where you can. This seems to help a lot with this issue.

What if I have a thousand icons on the page?

You might be wondering how rendering icons with JavaScript affects performance. We worried about this quite a bit while we were developing it and even took some special measures to make sure it was as fast as we could make it.

Our testing shows that for the typical number of icons most people use on a site the loading and rendering time is faster than web fonts. Your mileage may vary, of course, and we’d love to hear about your experience if this turns out not to be the case for you.

If you are using 1,000’s of icons on a page you might benefit from Symbols. This uses SVG with JS and the SVG sprites technique to reuse icons that are repeating. (Think rows of icons.)

Power Transforms NEW

Thanks to the power of SVG in Font Awesome 5, now you can scale, position, flip, & rotate icons arbitratily using the data-fa-transform element attribute. You can even combine them for some super-useful effects.

Power Transforms: Scaling

Power Transform scaling effects icon size without changing or moving the container. To scale icons up or down, use grow-# and shrink-# with any arbitrary value, including decimals. Units are 1/16em. For clarity in the example, we’ve added a background color on the icon so you can see the effect.

<div class="fa-4x">
  <i class="fas fa-magic" data-fa-transform="shrink-8" style="background:MistyRose"></i>
  <i class="fas fa-magic" style="background:MistyRose"></i>
  <i class="fas fa-magic" data-fa-transform="grow-6" style="background:MistyRose"></i>
</div>

Power Transforms: Positioning

Power Transform positioning effects icon location without changing or moving the container. To move icons up, down, left, or right, use up-#, down-#, left-#, and right-# with any arbitrary value, including decimals. Units are 1/16em. For clarity in the example, we’ve added a background color on the icon so you can see the effect.

<div class="fa-4x">
  <i class="fas fa-magic" data-fa-transform="shrink-8" style="background:MistyRose"></i>

Power Transform rotating & flipping effects icon angle and reflection without changing or moving the container. CSS Pseudo-elements

We’ve seen a lot of examples where adding markup like <i class="fas fa-user" /> allows you to easily add icons to any site. But what if changing the markup or HTML is not possible?

CSS has a powerful features known as Pseudo-elements. Font Awesome has leveraged the ::before pseudo-element to add icons to a page since the very beginning.

This method is not recommended. Use it when you have no other option.

Although you can use pseudo-elements with SVG and JS we do not recommend using this method. It does not provide as many usage options, is difficult to configure, and is the slowest method of using Font Awesome. (Painfully slow in some cases.)

A better alternative is to use normal <i> tags. Or if you really need pseudo-elements see if you can use web fonts with CSS instead. You'll have a much better experience.

You've been warned, friends.

How do they work with SVG

We’ve already learned that Font Awesome, through the JavaScript library, replaces <i> tags with inline SVG. It can also examine existing markup and through careful CSS styling add inline SVG for pseudo-elements. Let’s see how this works.

<script>
  FontAwesomeConfig = { searchPseudoElements: true };
</script>

<style>
  nav {
    display: flex;
    flex-direction: row;
    flex-wrap: nowrap;
    justify-content: space-between;
    align-items: center;
  }

  nav a {
    width: 30%;
    width: 15em;
    padding: 1em;
    color: rgba(255,255,255, 0.75);
    background-color: tomato;
    border-radius: 0.25em;
    text-decoration: none;
    text-align: center;
  }

  nav a:hover, nav a:focus {
    color: rgba(255,255,255, 1.0);
    text-decoration: none;
  }

  nav a::before {
    display: none;
  }

  nav .svg-inline--fa {
    margin-right: 0.5em;
  }

  .login::before {
    font-family: "Font Awesome 5 Solid";   content: "\f007";
  }

  .tps::before {
    font-family: "Font Awesome 5 Regular"; content: "\f1ea";
  }

  .twitter::before {
    font-family: "Font Awesome 5 Brands";  content: "\f099";
  }
</style>

<nav>
  <a href="#" class="login">Login</a>
  <a href="#" class="tps">TPS Reports</a>
  <a href="#" class="twitter">Twitter</a>
</nav>

Enable searching for pseudo-elements

Since searching for pseudo-elements requires DOM queries that could slow down performance for anyone who doesn’t need them they are disabled by default.

Enable searching for pseudo elements through the FontAwesomeConfig.

Order matters

Make sure you configure Font Awesome before the browser loads the main library.

<script>
  FontAwesomeConfig = { searchPseudoElements: true };
</script>
<script defer src="/static/fontawesome/fontawesome-all.js"></script>

Define a pseudo-element in CSS

There are four important parts to this.

  1. Set display: none; so that the actual pseudo element isn’t visible (we only need it for style reference)
  2. Set the font-family to Font Awesome 5 Solid, Font Awesome 5 Regular, Font Awesome 5 Light, or Font Awesome 5 Brands
  3. Set the content to the unicode value of one of our icons
  4. Set the pseudo-element to either ::before or ::after
.login::before {
  display: none;
  font-family: "Font Awesome 5 Solid";
  content: "\f007";
}

With jQuery

There are a few things that we’d like to tell you about with regard to integrating jQuery with the SVG version of Font Awesome.

Are you upgrading from version 4?

If this seems too complex you might consider switching to Web Fonts with CSS. Each technology in Font Awesome 5 has its benefits and integrating with jQuery can be a little daunting. Nesting the SVG tags inside of the <i> tags might also help you out.

Special considerations

We want to take a moment to explain a bit more how the JavaScript Font Awesome library works. Bear with us, this is important.

We start with an i tag 

<i class="fas fa-coffee" data-fa-mask="fas fa-circle"></i>

And end up with an svg tag 

<svg class="svg-inline--fa fa-coffee fa-w-20" aria-hidden="true" data-fa-i2svg="" data-prefix="fas" data-icon="coffee" role="img" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 640 512"><path fill="currentColor" d="M192 384h192c53 0 96-43 96-96h32c70.6 0 128-57.4 128-128S582.6 32 512 32H120c-13.3 0-24 10.7-24 24v232c0 53 43 96 96 96zM512 96c35.3 0 64 28.7 64 64s-28.7 64-64 64h-32V96h32zm47.7 384H48.3c-47.6 0-61-64-36-64h583.3c25 0 11.8 64-35.9 64z"></path></svg>
<!-- <i class="fas fa-coffee"></i> -->

How does this happen?

  1. It looks for DOM elements which resemble fas fa-coffee
  2. It figures out things like the prefix (fas), the icon (coffee), and anything else of interest
  3. It schedules the replacement of the <i> tag for the next convenient time
  4. When the browser is ready the <i> DOM element is replaced with a new <svg> element
  5. A comment is added below the new <svg> tag representing the original HTML for convenience

Why is this important?

This will not work because the <i> tag gets replaced 

<nav>
  <ul style="list-style: none; margin: 0; padding: 0;">
    <li><i class="fa fa-user fa-2x"></i></li>
  </ul>
</nav>

<script>
  document.addEventListener('DOMContentLoaded', function () {
    $('nav i').on('click', function () {  // the i tag will not exist once the icon is rendered
      alert('You will never see this');
    });
  });
</script>

This works because it is not dependent on the <i> tag 

<nav>
  <ul style="list-style: none; margin: 0; padding: 0;">
    <li><i class="fa fa-user fa-2x"></i></li>
  </ul>
</nav>

<script>
  document.addEventListener('DOMContentLoaded', function () {
    $('nav li').on('click', function () {
      alert('This works, though');
    });
  });
</script>

If you absolutely need to attach events to the icon you can use the data-fa-i2svg attribute but you need to allow for the dynamic creation of the svg element.

Attach the event to an ancestor and filter for the icon 

<nav>
  <ul style="list-style: none; margin: 0; padding: 0;">
    <li><i class="fa fa-user fa-2x"></i></li>
  </ul>
</nav>

<script>
  document.addEventListener('DOMContentLoaded', function () {
    $('nav').on('click', '[data-fa-i2svg]', function () {
      alert('You clicked the icon itself');
    });
  });
</script>

Avoid creating JavaScript variable references to icons

Since Font Awesome is dynamically replacing and modifying your icons any held reference to an icon will not work as expected.

This will not work and will leave you sad 

<nav>
  <ul>
    <li><i class="fa fa-user fa-2x"></i></li>
  </ul>
</nav>

<script>
  document.addEventListener('DOMContentLoaded', function () {
    var icon = $('nav i')

    $('nav li').on('click', '[data-fa-i2svg]', function () {
      icon.addClass('active');  // This has no effect, the original icon has been replaced
    });
  });
</script>

Changing icons by changing class

<button>Open up <i class="fa fa-angle-right"></i></button>

<script>
  document.addEventListener('DOMContentLoaded', function () {
    $('button').on('click', function () {
      $(this)
        .find('[data-fa-i2svg]')
        .toggleClass('fa-angle-down')
        .toggleClass('fa-angle-right');
    });
  });
</script>
